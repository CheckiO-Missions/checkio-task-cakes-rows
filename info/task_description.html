The cakes are laying on the floor, and all cakes are round. Our robots can't help but try to figure out a pattern behind the cakes' disposition. Some cakes form rows and we want to count these rows. A row is a sequence of three or more cakes if we can draw a straight line through its centers. The greater row takes up the smaller rows. So if we have a row with 4 cakes, then we have only one row (not 4 by 3).

The cakes' location is represented as a list of coordinates. A coordinate is a list of two integers. You should count the rows.

Input: Сoordinates as a list of lists with two integers.

Output: The quantity of rows as an integer.

Example: 

checkio([[3, 3], [5, 5], [8, 8], [2, 8], [8, 2]]) == 2
checkio([[2, 2], [2, 5], [2, 8], [5, 2], [7, 2], [8, 2], [9, 2], [4, 5], [4, 8], [7, 5], [5, 8], [9, 8]]) == 6

How it is used: This is an example of the image recognition. If to be more careful -- it's the pattern recognition. This concept can be useful for the game mechanics or if you want to write a bot for games.

Precondition: 0 < |coordinates| < 20
∀ x,y ∈ coordinates : 0 ≤ x,y ≤ 10
